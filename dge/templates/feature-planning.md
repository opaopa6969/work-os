# DGE Template: 新機能企画

## 概要
新機能を実装する前に要件の穴を見つける。

## 推奨キャラクター
今泉 (前提確認) + ヤン (簡素化) + 僕 (scope 縮小)

## Scene 構成

### Scene 1: ユーザーストーリー
```
focus: 誰が、何のために、何をするか
今泉: 「そもそもこの機能、誰が欲しいって言ったんですか？」
ヤン:  「1 番大事なユースケースだけ教えて。残りは後で。」
僕:   「...ユーザー 3 人に聞けばいいんじゃ...」

抽出すべき Gap:
- ターゲットユーザー
- 解決する課題
- 既存の代替手段
- 成功指標 (KPI)
```

### Scene 2: UI/UX フロー
```
focus: ユーザーが画面で何をするか
今泉: 「ボタンを押したら何が起きるんですか？」
ヤン:  「画面遷移 3 つ以内で終わらないと使わない。」
僕:   「...モーダル 1 枚でいいんじゃ...」

抽出すべき Gap:
- 画面遷移図
- 入力フォームの項目
- エラー時の表示
- ローディング状態
```

### Scene 3: 技術的制約
```
focus: 実装の難易度、依存、リスク
今泉: 「これ作るのに何日かかるんですか？」
ヤン:  「既存のコードで 80% できない？」
僕:   「...新しいライブラリ入れたくない...」

抽出すべき Gap:
- 技術的な依存 (新ライブラリ？)
- 既存コードへの影響
- 工数見積もり
- リスク
```

### Scene 4: MVP Scope
```
focus: 最小限の実装範囲
今泉: 「全部必要ですか？ 半分でいいんじゃ？」
ヤン:  「1 つだけ作るなら何？」
僕:   「...全部やろうとすると心が折れる...」

抽出すべき Gap:
- must-have vs nice-to-have
- v1 scope vs v2 backlog
- 削っても成立する最小セット
```
